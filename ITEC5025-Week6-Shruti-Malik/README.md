## Dataset: 100,000-Patient EMRBots Synthetic Dataset

### Source
Synthetic patient data from [EMRBots](http://www.emrbots.org/) — a fictional dataset designed for educational use. **No real patient data is used.**

### Tables

| File | Description | Rows |
|------|-------------|------|
| `PatientCorePopulatedTable.txt` | Patient demographics (gender, DOB, race, language, poverty %) | ~100,000 |
| `AdmissionsCorePopulatedTable.txt` | Hospital admission records | ~500,000+ |
| `AdmissionsDiagnosesCorePopulatedTable.txt` | ICD-10 primary diagnoses per admission | ~500,000+ |
| `LabsCorePopulatedTable.txt` | CBC, metabolic, and urinalysis lab results | Very large |

### Cleaning Steps Applied
- Stripped whitespace from all string columns
- Dropped rows with null or empty `PatientID`
- Parsed all date columns to ISO-8601 format (`YYYY-MM-DD HH:MM:SS`)
- Converted numeric columns (coerced invalid values to NaN)
- Removed exact duplicate rows
- Renamed columns to `snake_case`

### Step 4 – Load and Clean the Patient Dataset

```powershell
python load_patient_data.py
```

This reads directly from `100000-Patients.zip` and saves cleaned CSVs to the `data/` folder.

## Database Setup (MySQL)

### Step 5 – Run the SQL Script

Open MySQL Workbench (or the MySQL command line) and run:

```sql
SOURCE chatbot_responses.sql;
```

Or from the command line:

```powershell
mysql -u root -p < chatbot_responses.sql
```

This will:
1. Create the `hypotify_chatbot` database
2. Create the `chatbot_responses` table with 20 pre-loaded responses
3. Create the `patient_demographics` table (mirrors the 100k dataset schema)
4. Insert 5 sample patient rows to demonstrate correct syntax
